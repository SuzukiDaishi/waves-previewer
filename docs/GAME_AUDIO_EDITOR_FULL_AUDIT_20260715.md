# NeoWaves 全体監査 — 理想機能の全列挙と現状の弱点・UX問題・バグの洗い出し (2026-07-15)

本書は「Volt (Inamon's Sound) / Sound Forge / iZotope RX を代替するゲーム特化リスト形式サウンドエディタ」という目標に対して、

1. **一から作る場合に入れるべき機能の全列挙**(現状の実装ステータス付き)
2. **現状の弱い機能・ギャップ**
3. **バグ全リスト**(確認済み+高確度のコード上の問題)
4. **UX問題全リスト**
5. **優先度付きロードマップ**

をまとめたものである。コードベース全体(src/ 約113k行・約140ファイル)、`docs/`、`CHANGELOG.md`、`tests/` を横断調査した結果に基づく。根拠はすべて `file:line` または doc 参照で示す。

凡例: ✅ 実装済み(Solid) / 🟡 部分実装・弱い / ❌ 未実装

## 1. エグゼクティブサマリ

**現在地**: リストビュー(100万ファイル対応)、ループ工房、VST3/CLAP アウトオブプロセスホスティング、エフェクトグラフ、AI(Whisper / All-In-One / Demucs)、CLI は既に競合ツールに無い水準で **Solid**。一方で:

- **RX 代替(スペクトル修復)が最も遠い**: スペクトル編集は「矩形 Mute + Warp」の2操作のみ。heal/inpaint、ブラシ、De-click/De-noise 系が無い。
- **Forge 代替の穴**: エディタ内 paste-insert、無音挿入、DC除去、位相反転、ディザ、スクラブ再生が無い。
- **モニタリングが致命的に弱い**: 再生が全chモノラル平均のみで、ステレオ定位を確認できない(ゲームSE制作では要確認事項の筆頭)。
- **UXの構造問題**: エラー通知層が存在しない(F12デバッグ窓のみ)、修飾なし単キーで破壊的編集が発火、キーマップ変更不可、docs/実装間の仕様矛盾。
- **データ損失リスクが3件**: Windows 非アトミック保存、ゲインのハードクリップ、未保存編集の無警告終了。

**最優先**は「データ損失系3件の封じ込め」と「通知(toast)層の導入」。機能追加より先にここを塞ぐべき。

## 3. 弱い機能・ギャップ一覧(要約)

上表の 🟡/❌ を柱ごとに要約すると:

1. **スペクトル編集**: Mute+Warp のみ。RX代替を名乗るには Heal/ブラシ/De-click/De-noise/De-hum/De-clip が必要(§2.3)。
2. **モニタリング**: モノラル再生のみ・再生中メーター無し・ch solo無し(§2.6)。
3. **波形編集の基本操作の穴**: paste-insert / 無音挿入 / DC除去 / 位相反転 / ペンシル / ディザ(§2.2)。
4. **プラグイン**: リアルタイムプレビュー無し・プリセット無し・マネージャ無し(§2.5)。
5. **ゲームエンジン連携**: 標準チャンク頼みで、Wwise/FMOD/Unity 向けの明示的な出口が無い(§2.4)。
6. **WORLD**: F0編集のみで包絡/非周期性が編集不可(§2.3)。
7. **バッチloudnormのGUI導線欠如**: CLIにしかない(§2.1)。
8. **UXインフラ**: 通知層・キーマップ・終了ガード・i18n(§2.7)。

## 5. UX問題全リスト

### 5.1 キーボード/ショートカット

1. **修飾なし単キーで破壊的編集が発火**: `C`=範囲削除+結合、`T`=トリムが確認なしで即実行(`input_ops.rs:275-290`)。誤爆リスク大。修飾キー化または確認/即undoトースト必須。
2. **Split キーが無い**: 波形エディタ最重要操作の分割が存在せず、`S` はビュー切替に消費。
3. **全ショートカットがハードコードでリバインド不可**: keymap テーブルも設定UIも無し。Forge/RXユーザーはこれだけで離脱し得る。
4. **アプリ内ショートカット一覧/Helpが無い**: メニューは File/Export/List/Tools のみ。Edit メニューも Help メニューも無く、キー一覧は日本語mdファイルのみ。
5. **コンテキストで同一キーの意味が変わる**: `P`=オートプレイ(リスト)/ループ終端(エディタ)、`R`=regex/ゼロクロススナップ。フォーカスルーティングがヒューリスティックで、どちらが発火するか予測しにくい。
6. 数字キーのマッピングが直感に反する(`2`→1/2、`9`→1/9)+`1`=末尾バグ(バグ#6)。一般的な「1..9=10..90%」への変更を推奨。
7. エディタで `Home/End`(先頭/末尾)、`+/-`ズーム、`[`/`]`、zoom-to-selection、Escでプレビューキャンセル、が無い。
8. エディタ内の音声 Cut/Copy/Paste ショートカットが無い(Ctrl+C/V はリスト専用)。

### 5.2 エディタ

9. **選択がマウス専用**: Shift+矢印での選択伸縮、select-all/to-end、キーボードでのゼロクロス移動が無い(矢印はズーム/シークに割当済み)。
10. **ホイール=ズームが標準と逆**(Forgeは素のホイール=スクロール)。`docs/UX.md:83` 自身が Ctrl+wheel を提案しており設計とも乖離。環境によりホイールズーム不安定(バグ#15)。
11. **17ツールが1個の ComboBox に押し込み**(`editor.rs:7001-7026`)。ツールバー・アイコン・ツールショートカット無し。切替コスト高。
12. **ツール切替で未適用プレビューを無言破棄**+再生停止(`editor.rs:7027-7069`)。
13. **非破壊状態の4層構造(pending→editing→committed→saved)が読みにくい**: ツール毎に同じ3行ステータスが繰り返され認知負荷が高い(`editor.rs:7117-7159`)。
14. **長時間処理が全画面ブロッキングオーバーレイ**: Pitch/Stretch 中はリスト閲覧も不可。バックグラウンド化+topbar活動スロットへの統合を推奨(`docs/KNOWN_ISSUES.md:10`)。
15. Undo/Redo が Inspector 内ボタンとホットキーのみで、メニューに無い。

### 5.3 リスト

16. **単クリックが「選択+即ロード/再生」を兼ねる**: 選択したいだけで音が鳴る。各セルが個別に click/double-click 判定を再実装しており挙動も不揃い(`list.rs:1317-1326`)。

20. コンテキストメニューに「Open in Editor」「Reveal in Folder」「Select All/None」が無い。
21. 1万〜3万件でメタ値(LUFS等)の充足が遅く、`...` 以外の進捗表現が無い(`docs/KNOWN_ISSUES.md:33`)。

### 5.4 モダリティ/ワークフロー

# 既知の注意点 / 制限

- エラー・警告はトップバー下の通知トーストに表示されます（クリックで消去、詳細は F12 デバッグウィンドウ）。セッション保存失敗・エクスポート失敗・タブ上限・リサンプラ品質フォールバックなどが対象です。
- 未保存編集（dirty タブ / pending gain）がある状態でウィンドウを閉じると確認ダイアログが出ます。Ctrl+W も dirty タブでは Leave Editor 確認を経由します。
- リストの Ctrl+C/V が Windows で反応しない問題（docs/CLIPBOARD_HOTKEY_ISSUE_20260201.md）は、egui の `Event::Copy` / `Event::Paste` を処理する現行実装（clipboard_ops.rs）で解決済みです。

- Windows で `NeoWaves.exe` を実行中のまま再ビルドすると「アクセスが拒否されました（OS error 5）」で失敗します。実行中のアプリを終了してからビルドしてください。
- README などの日本語が文字化けする場合があります。全ファイルを UTF-8 (BOM 無し) で保存してください。UI 側は Windows で Meiryo/Yu Gothic/MS Gothic を動的読み込みし、豆腐文字を軽減します。
- ループ範囲編集（Inspector > LoopEdit、K/P ショートカット、L での適用）は Undo/Redo に対応済みです。
- 再生はマルチチャンネル対応です。ソース ch 数が出力 ch 数以下ならそのままマッピング、超える場合は余剰チャンネルを平均で折り畳みます（チャンネルを黙って破棄しません）。ツールプレビュー（Fade/Gain/Normalize/Loudness/Reverse/EQ/Comp/Gate 等）もステレオのまま試聴されます。※長尺クリップの Pitch/Stretch ヘビープレビューは設計上モノラルです。
- エディタの M/S（チャンネル別 mute/solo）はモニタリング専用です。フォールドダウン時は可聴チャンネルのみを平均します。エフェクトグラフやエクスポートには影響しません。
- トップバーの出力メーターは再生中に出力チャンネル毎の RMS+ピークホールドを表示します。オーディオコールバックが動いていない場合（デバイス無し等）のフォールバック値はモノラル1本のままです。
- リストの列幅はグローバル prefs に保存されます（マシン/DPI 毎）。プロジェクト（.nwsess）別の列幅と列の並べ替えは未対応です。
- キーマップは一元テーブル化済みですが、キー割当の変更 UI は未実装です（Help > Keyboard Shortcuts は閲覧のみ）。
- エディタ内オーディオクリップボード（Ctrl+C/X/V、Ctrl+Shift+V=ミックス、Ctrl+Alt+V=クロスフェード挿入）はアプリ内専用です（OS クリップボード経由の他アプリとの音声交換は非対応）。
- チャンネルスコープ編集（Custom チャンネルビュー時の範囲編集マスク）は軽量プレビュー（Gain/Normalize/Fade/Invert/DC）には反映されますが、長尺ファイルのオーバービュー型プレビューは全チャンネル表示のままです（Apply は正しくマスクされます）。
- Spectral Brush / Heal / De-noise は簡易実装です: Heal は左右コンテキストの線形補間+位相伝搬のため、非定常な内容の長い区間では「凍った」音になります。ステレオの位相相関は保証されません。De-noise のプロファイルはサンプルレート変更で失効します（学習し直してください）。
- De-click の修復は 3 次エルミート補間です（AR/LPC 外挿は将来のアップグレード候補）。2 ms を超えるトランジェントはコンテンツと見なして修復しません。
- Plugin FX の Auto preview は毎回全体をレンダリングします（選択範囲+プリロールの部分レンダリングによる高速化は未実装）。ワーカープロセスは都度起動のため、重いプラグインでは応答が遅くなります。プリセット/A/B スロットと Auto preview 設定はセッション限りです（プロジェクトへは保存されません）。
- Find Duplicates の類似判定はペア総当たり（長さが 10% 超違うペアは除外）です。数千件規模までを想定しており、それ以上は時間がかかります。開始オフセットが異なる同一素材（頭出し違い）は検出しません。
- Light テーマ: エディタのオーディオキャンバス（波形レーン・スペクトログラム LUT・ミニメーター）は両テーマとも意図的にダークのままです（DAW 流儀）。リスト/トップバーの手描きウィジェットはテーマ対応済み。
- BWF は bext のみ対応です（iXML の編集は未対応）。UMID とタイムリファレンスはゼロで書き込みます。
- i18n（日英切替）はユーザー判断で見送りました（キーマップ/一元テーブル等、構造はその準備を兼ねています）。
- TPDF ディザは 16bit 整数 PCM のみ対象です（24bit と lossy コーデックは対象外）。プレビュー/リスト描画用の量子化シミュレーションはディザしません。
- Invert Polarity の部分適用は境界をスムージングしません（ゼロクロススナップ併用を推奨）。
- 一括検査(Inspect Files)は silence チェック有効時に 1 ファイル 1 デコードを行うため、数千件規模では時間がかかります(ワーカー≦4本・低優先度で UI はブロックしません)。
- Normalize Loudness は、対象ファイルがエディタタブで開かれている場合は破壊的なタブ編集(エディタ側 Undo)になります。それ以外は pending gain(リスト Undo)です。キャンセル時、既にタブへ適用された編集はエディタの Undo で戻してください。
- リサンプルは rubato（sinc/FFT）による品質 SRC を使用します。rubato が失敗した場合のみ線形補間へフォールバックし、その際は警告トーストで通知されます。Pitch/Stretch のオフライン前段と LUFS 用 48 kHz 変換は現状も線形です。
- Wave 列の行高は列幅変化に追随して 1 フレーム遅延で更新されます（計測→次フレーム適用）。
- 非対応の出力フォーマットやデバイスで CPAL が開けない場合があります。出力デバイスは設定から選択できます。
- PitchShift/TimeStretch はファイルサイズやパラメータにより処理時間がかかることがあります。処理中はUIをブロックするローディングカバーを表示します。
- Windows で signalsmith-stretch を使う場合、LLVM（libclang.dll）が必須です。`LIBCLANG_PATH` の設定が必要な環境があります。

■ ラウドネス（LUFS）に関する注意
- 本実装は 48 kHz へ簡易リサンプル後、BS.1770 のK-weighting／ゲートで LUFS(I) を算出します。厳密な係数再設計は今後の改善項目です。
- LUFS(I) は Gain 変更に即時追従するため表示上は `base + gain` の近似を出します。数百ms後に厳密再計算で確定します。
- −70 LUFS 近傍ではゲート通過/不通過が反転し、近似と厳密値が一時的に異なる場合があります（最終的には厳密側へ収束）。
- マルチチャンネル重みは現状すべて 1.0（LFE除外は未対応）。将来、サラウンド +1.5 dB の重み付けに対応予定です。

■ オーバーレイ（緑）と背景の厳密一致
- ズーム最遠（Aggregated）で列が合わない問題を修正済み（px列ロック+可視窓比率マッピング）。
- 端列（最初/最後）の 1px で ±1 サンプルの丸め誤差が出る場合があります。実害はありませんが、必要に応じて端列のスナップを追加予定です。
- 高ズーム時にスペクトル系レーン（Spec/Log/Mel/Tempogram/Chromagram/World）が波形レーンに対して最大1ホップ分ずれる問題は修正済みです（列ごとの小数フレームマッピング）。

■ エディタの時間ズーム/パンについて（改善済み・注意点）
- 取得経路を強化しました：`raw_scroll_delta` に加えて低レベルイベント `Event::Scroll` / `Event::Zoom`（ピンチ）も併用し、環境差でホイールが 0 になるケースを軽減しています。
- キャンバス上のポインタ位置を矩形判定で検出し、hover 取りこぼしで無効化される問題を回避しています。
- それでも環境依存で挙動が弱い場合があります（高分解能ホイール、OS のジェスチャ設定等）。
  - 対策: キャンバス上にポインタを置いた状態で操作／トラックパッドはピンチを使用／Shift+ホイールでパン。
  - デバッグ（開発ビルド）: 操作時に `stderr` へ `wheel_raw=... pinch=...` を出力します。値が常に 0 の場合はプラットフォーム統合層で吸収されている可能性があります。
  - 再現ケースがあれば issue へログと環境（OS/デバイス）を添えてご報告ください。

パフォーマンス関連（大規模リスト）
- 1万〜3万件規模のリストは仮想化で十分軽快に動作しますが、初回のメタ計算（RMS/サムネ）はバックグラウンドで順次行われるため、すべての行に値が出揃うまでに時間がかかります。
- さらなる高速化を行う場合は以下を検討してください:
  - 可視範囲優先のメタ生成キュー（優先/通常の二段階キュー）。
  - 文字列の構築結果（表示名/親パス文字列）のキャッシュ化。
  - サムネbinsの上限を 128–256 程度に固定して再描画コストを一定に。
